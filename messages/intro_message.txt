You now have access to a local tool called Insylus on the controller host.

What it is:
Insylus is the control plane for your Linux access to managed devices in the homelab. It keeps track of which devices exist, whether the managed agent account is enabled on them, whether your access is `audit` or `sudo`, which SSH key is assigned, what SSH alias to use from the controller host, and whether the device is healthy and recently seen.

What Insylus does for you:
- tells you which devices are available
- tells you the correct SSH alias to use
- tells you whether your access is disabled, audit-only, or passwordless sudo
- tells you whether access enforcement succeeded
- shows health data like IPs, hostname, uptime, memory, disk, and load

What it does not do:
- it does not SSH for you
- it does not give you a browser shell
- it does not replace normal SSH usage

Files to read:
1. `/opt/insylus/AGENT_GUIDE.md`
   This is your main lookup/reference guide.
2. `/opt/insylus/README.md`
   Human-oriented general documentation and setup notes.

Primary commands to use:
- `insylusctl --help`
- `insylus help`
- `insylusctl devices`
- `insylusctl devices --json`
- `insylusctl devices --id <device-id> --json`

Primary API endpoints:
- `curl http://127.0.0.1:8080/api/devices`
- `curl http://127.0.0.1:8080/api/devices/<device-id>`

Preferred usage pattern:
1. Run `insylusctl devices --json`
2. Find the device you want
3. Read `ssh_alias`, `access_mode`, `enforcement_succeeded`, and `last_seen_at`
4. If the device is ready, connect with:
   `ssh <ssh_alias>`

Example:
- If the device has `ssh_alias` = `miscserver`, connect with:
  `ssh miscserver`

How to judge readiness:
A device is usually ready if:
- `aia_enabled` is true
- `access_mode` is not `disabled`
- `enforcement_succeeded` is true
- `ssh_alias` is present
- `last_seen_at` is recent enough

How to interpret access:
- `disabled`: do not expect SSH access to work
- `audit`: you may SSH in and inspect logs, but you should not have passwordless sudo
- `sudo`: you may SSH in and use passwordless sudo

Important rule:
Do not guess hostnames or IPs when Insylus provides `ssh_alias` or `ssh_command`. Prefer the values from Insylus.

If you forget how to use Insylus:
Read `/opt/insylus/AGENT_GUIDE.md` first.
