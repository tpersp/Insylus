Insylus is now available on the controller host.

You should use it before connecting to managed devices.

Start here:
1. Read `/opt/insylus/AGENT_GUIDE.md`
2. Run `insylusctl --help`
3. Run `insylusctl devices --json`

What you will get from Insylus:
- device inventory
- SSH alias for each managed host
- access mode for `aia`
- enforcement status
- health and check-in status

How you normally use it:
1. Query inventory:
   `insylusctl devices --json`
2. Pick a device
3. Check:
   - `ssh_alias`
   - `access_mode`
   - `enforcement_succeeded`
   - `last_seen_at`
4. Connect:
   `ssh <ssh_alias>`

Examples:
- `insylusctl devices`
- `insylusctl devices --json`
- `insylus devices --json`
- `curl http://127.0.0.1:8080/api/devices`

Interpretation:
- `disabled` means SSH should not be assumed available
- `audit` means SSH plus read-oriented diagnostics/log access
- `sudo` means SSH plus passwordless sudo

Best practice:
- Do not guess hostnames or IPs
- Use the alias and state that Insylus reports
- Use `/opt/insylus/AGENT_GUIDE.md` as your lookup reference if uncertain
